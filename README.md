# 验证码提取器

一款专为 Windows 11/10 设计的后台无感运行的验证码提取工具。它能够实时监控并劫持系统右下角的弹窗通知（如 QQ、微信、Chrome、邮件等），智能提取其中的验证码并自动复制到剪贴板，彻底解放双手。

## ✨ 核心特性

* 👻 **完全无感**：无主界面，无弹窗打扰。启动后直接潜伏于系统托盘，默默为你工作。
* ⚡ **双核提取引擎**：
    * **本地正则引擎 (默认)**：毫秒级响应，无需联网，智能规避时间/日期等干扰项，精准提取 4~6 位验证码。
    * **云端 AI 引擎**：接入火山引擎大模型 (Volcengine)，应对极其复杂或非标准格式的验证码通知。
* 🚀 **绿色自启**：一键开启静默开机自启，采用原生 PowerShell 脚本创建快捷方式，杜绝修改注册表引发的杀毒软件报毒。
* 🔔 **即时反馈**：提取成功后自动播放系统提示音，并支持 Windows 原生气泡通知反馈。

## ⚙️ 配置文件 (config.json)

在程序首次保存配置后，会在同级目录下生成 __config.json__。你也可以手动创建该文件并填入以下默认内容：

```json
{
    "LLM_API_KEY": "",
    "LLM_MODEL": "",
    "LLM_BASE_URL": "https://ark.cn-beijing.volces.com/api/v3",
    "EXTRACT_MODE": "REGEX"
}
```

### 字段说明：
- __EXTRACT_MODE__: 核心提取模式。__REGEX__ 表示本地正则匹配（推荐，速度最快），__LLM__ 表示云端大模型提取。
- __LLM_API_KEY__: 火山引擎 API 密钥，仅在 LLM 模式下需要。
- __LLM_MODEL__: 大模型接入点 ID（Endpoint ID）。
- __LLM_BASE_URL__: API 请求地址，默认使用火山引擎 V3 接口。

## 🚀 运行与使用

### 1. 源码运行
直接在终端执行脚本即可启动：
```bash
python getCodeByWinInform_Lite2.0.py
```
启动后，程序会自动隐藏到 Windows 右下角的**系统托盘**。

### 2. 引擎配置
右键点击托盘图标，选择 **「引擎配置」**，你可以：
* 切换 **本地正则匹配**（推荐，速度最快）或 **云端大模型提取**。
* 如果使用云端模型，请填入你的火山引擎 API Key 及模型接入点（Model ID）。
* 开启/关闭 **开机静默启动**。

配置保存后，会在程序同级目录下自动生成一个 `config.json` 文件进行持久化存储。
或者你可以直接在`config.json`文件中进行编辑

## 📦 极致打包指南

若要将 Python 脚本打包为单文件 `.exe` 以便分发，请使用 `PyInstaller`，并**务必**按照以下命令执行，以确保软件体积最小且能正确加载托盘图标：

### 准备工作
请在脚本同级目录下准备一张名为 `new-email-verification.png` 的图标文件（建议使用透明底色的 PNG）。

### 执行打包命令
```bash
pyinstaller --noconsole --onefile --noupx --add-data "new-email-verification.png;." getCodeByWinInform_Lite2.0.py
```

**参数说明：**
* `--noconsole`: 隐藏黑色 CMD 控制台窗口，实现真正的后台运行。
* `--onefile`: 打包为独立的单体 `.exe` 文件。
* `--noupx`: 禁用 UPX 压缩，防止底层 `winsdk` 通知劫持组件被杀毒软件误杀。
* `--add-data`: 将图标文件一并封装进 EXE 内部。

## ⚠️ 常见问题排查 (FAQ)

**Q1：为什么程序运行后什么都没发生？**
A：本程序是纯后台工具，启动后请去桌面右下角的**任务栏托盘（隐藏图标列表）**中寻找它。首次启动时会弹出系统气泡提示。

**Q2：为什么收到了验证码通知，但没有自动复制？**
1.  **系统权限拦截**：请前往 Windows 设置 -> 系统 -> 通知，确保「允许应用访问通知」的权限已开启。
2.  **通知来源不在监控列表中**：目前代码中默认监控的来源为 `["QQ", "邮件", "Mail", "Outlook", "微信", "Chrome"]`，如果你的通知来自其他应用，请在代码的 `TARGET_APPS` 列表中自行添加。

**Q3：点击保存配置时卡死？**
A：当前版本已通过多线程与队列 (`queue`) 通信彻底重构了底层逻辑，完美解决了 GUI 假死与死锁问题。请确保你使用的是最新的 `Lite 2.0` 版本代码。
